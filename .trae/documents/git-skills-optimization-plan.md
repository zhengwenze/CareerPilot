# Git 技能优化计划

## 现有技能评估

### 当前技能清单

1. **dual-repo-publish** - 双仓库发布流程
2. **git-commit-convention** - Conventional Commits 规范

### 现有技能优势

✅ **dual-repo-publish**:
- 清晰的双远程仓库配置（origin/Gitee/master, github/GitHub/main）
- 完整的安全检查流程（祖先验证、空提交阻止）
- 详细的失败处理策略
- 明确的验证步骤

✅ **git-commit-convention**:
- 完整的 Conventional Commits 规范
- 详细的类型定义和范围规则
- 清晰的提交流程决策树
- 中英双语支持

## 需要优化的方面

### 1. 技能间协作不够明确

**问题**: 两个技能之间如何协作使用不够清晰

**建议改进**:
- 在 dual-repo-publish 中明确引用 git-commit-convention 的具体方式
- 添加技能组合使用示例

### 2. 缺少常见场景处理

**问题**: 以下常见场景未覆盖

- 部分推送成功后的恢复流程
- 远程仓库认证失败处理
- 大文件提交前的检查
- 敏感信息泄露检查

### 3. 缺少交互式提交流程

**问题**: 当前流程是全自动的，缺少与用户的确认环节

**建议改进**:
- 添加提交前确认步骤
- 提供提交信息预览和修改机会
- 支持分批提交交互

### 4. 验证机制可以加强

**问题**: 推送后的验证仅限于哈希比对

**建议改进**:
- 添加远程仓库可访问性预检
- 验证推送后的文件完整性
- 检查分支保护规则

## 具体优化建议

### 优化 1: 增强 dual-repo-publish 技能

#### 添加预检步骤
```markdown
## Pre-flight checks

在正式执行前，进行以下检查：

1. 网络连通性检查
   - 测试与 Gitee 和 GitHub 的连接
   - 检查是否需要代理

2. 认证状态检查
   - 验证 Git 凭证是否有效
   - 提示用户处理过期凭证

3. 仓库状态检查
   - 确认本地分支与远程追踪分支一致
   - 检查是否有未完成的合并或变基
```

#### 添加交互确认流程
```markdown
## Interactive confirmation

在创建提交前，向用户展示：

1. 变更摘要
   - 修改的文件列表
   - 新增/删除/修改的行数统计
   
2. 建议的提交信息
   - 基于变更自动生成的提交信息
   - 询问用户是否确认或修改

3. 推送预览
   - 显示将要推送到的远程仓库
   - 显示目标分支
```

#### 增强错误恢复
```markdown
## Recovery procedures

### 场景 1: Gitee 推送成功，GitHub 失败

1. 记录当前状态（部分同步）
2. 分析 GitHub 推送失败原因
3. 提供重试选项
4. 如需强制同步，提供安全的回滚方案

### 场景 2: 认证失败

1. 识别具体哪个远程仓库认证失败
2. 提供凭证更新指引
3. 支持临时凭证输入
```

### 优化 2: 创建新的辅助技能

#### 建议新增: git-pre-commit-check

```markdown
# Git Pre-commit Check

在提交前执行各种检查：

## 检查项

1. 敏感信息扫描
   - API 密钥
   - 密码
   - 私钥文件

2. 大文件检查
   - 超过 10MB 的文件警告
   - 建议使用 Git LFS

3. 代码规范检查
   - 运行 lint
   - 检查文件格式

4. 测试检查
   - 提醒运行测试
   - 检查测试覆盖率
```

#### 建议新增: git-conflict-resolution

```markdown
# Git Conflict Resolution

处理合并冲突的指导技能：

## 冲突类型识别

1. 内容冲突
2. 重命名冲突
3. 删除冲突

## 解决步骤

1. 识别冲突文件
2. 分析冲突原因
3. 提供解决建议
4. 验证解决方案
```

### 优化 3: 改进技能文档结构

#### 统一的前置条件检查清单

在每个 Git 技能开头添加：

```markdown
## Prerequisites Checklist

- [ ] Git 版本 >= 2.30
- [ ] 远程仓库已配置
- [ ] 网络连接正常
- [ ] 工作区干净或无重要未保存更改
```

#### 添加快速参考卡片

```markdown
## Quick Reference

| 场景 | 命令/操作 |
|------|----------|
| 检查状态 | `git status --short` |
| 查看远程 | `git remote -v` |
| 验证同步 | `git rev-parse HEAD origin/master github/main` |
```

## 总结

现有 Git 技能已经具备了良好的基础，主要需要在以下方面加强：

1. **健壮性**: 添加更多预检和错误恢复机制
2. **用户体验**: 增加交互确认环节
3. **完整性**: 补充提交前检查等辅助技能
4. **文档**: 改进技能间的协作说明

这些优化将使 Git 工作流更加可靠和用户友好。
