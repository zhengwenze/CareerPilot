# 本地环境安装清单

适用对象：`MacBook Air M4`、`macOS 15`、从 0 开始搭建《职点迷津》开发环境

这份清单只解决一件事：让你的电脑具备“能开始开发《职点迷津》”的最低环境。

---

## 一、先说结论

你这台电脑可以开发《职点迷津》。

当前已经有的环境：

- `Node.js`
- `npm`
- `Docker CLI`

当前还缺的关键环境：

- `Docker Desktop` 需要真正启动
- `uv`
- `pnpm`
- `Python 3.11`
- `Homebrew`（推荐安装，后面装 OCR 会更方便）

---

## 二、安装顺序

按下面顺序装，最省事：

1. 安装 `Homebrew`
2. 安装并启动 `Docker Desktop`
3. 安装 `uv`
4. 用 `uv` 安装 `Python 3.11`
5. 启用 `pnpm`
6. 可选安装 `OCRmyPDF`

---

## 三、一步一步安装

### 1. 安装 Homebrew

作用：Mac 上最好用的包管理工具，后面装 OCR 和一些系统依赖会方便很多。

官方命令：

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

装完后验证：

```bash
brew --version
```

如果能看到版本号，就说明成功了。

官方来源：[Homebrew 官网](https://brew.sh/zh-cn/)

### 2. 安装并启动 Docker Desktop

作用：本地跑 `PostgreSQL`、`Redis`、`MinIO` 这些基础服务。

你现在电脑上只有 Docker 命令行，还没有真正启动 Docker 服务，所以这一步必须做。

安装方法：

1. 打开官方页面，下载 `Docker Desktop for Mac with Apple silicon`
2. 安装后打开 Docker Desktop
3. 等右上角 Docker 图标稳定

安装好后验证：

```bash
docker info
docker run --rm hello-world
```

如果第二条命令成功输出欢迎信息，就说明 Docker 已经能用了。

官方来源：[Docker Desktop for Mac](https://docs.docker.com/desktop/setup/install/mac-install/)

### 3. 安装 uv

作用：这是我们后端 Python 项目的依赖管理工具，也可以顺手帮你装 Python 版本。

官方命令：

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

安装后关闭并重新打开终端，再验证：

```bash
uv --version
```

官方来源：[uv 安装文档](https://docs.astral.sh/uv/getting-started/installation/)

### 4. 用 uv 安装 Python 3.11

为什么不是直接用你现在的 Python？

因为你本机现在是 `Python 3.13`，而《职点迷津》这种 AI 后端项目，用 `Python 3.11` 会更稳，兼容性更好。

安装命令：

```bash
uv python install 3.11
```

查看已安装 Python：

```bash
uv python list
```

如果你后面要把项目固定到 3.11，在项目目录里执行：

```bash
uv python pin 3.11
```

官方来源：[uv 安装 Python 文档](https://docs.astral.sh/uv/guides/install-python/)

### 5. 启用 pnpm

作用：这是前端项目的包管理工具。

你已经有 `Node.js`，所以最简单的方法是直接启用 `corepack`。

命令：

```bash
corepack enable pnpm
corepack use pnpm@latest-10
```

验证：

```bash
pnpm --version
```

如果上面不行，也可以用备用命令：

```bash
npm install -g pnpm@latest-10
```

官方来源：[pnpm 安装文档](https://pnpm.io/installation)

### 6. 可选安装 OCRmyPDF

作用：处理扫描版简历 PDF。这个不是第一分钟必须安装，但你后面做到“简历解析 OCR 兜底”时很有用。

如果已经装了 Homebrew，可以直接执行：

```bash
brew install ocrmypdf
```

如果需要更多语言包：

```bash
brew install tesseract-lang
```

验证：

```bash
ocrmypdf --version
tesseract --version
```

官方来源：  
[OCRmyPDF 安装文档](https://ocrmypdf.readthedocs.io/en/latest/installation.html)  
[Tesseract 安装文档](https://tesseract-ocr.github.io/tessdoc/Installation.html)

## 四、哪些东西现在不用装

下面这些，现阶段可以先不单独装：

- `PostgreSQL`
- `Redis`
- `MinIO`
- `psql`
- `redis-server`

原因很简单：我们优先用 `Docker` 跑这些服务，不需要先把它们装到系统里。

---

## 五、全部装完后，一次性检查

把下面命令逐条执行一遍：

```bash
brew --version
docker info
uv --version
uv python list
pnpm --version
node --version
npm --version
python3 --version
```

理想状态：

- `brew` 有版本号
- `docker info` 不报错
- `uv` 有版本号
- `uv python list` 能看到 `3.11`
- `pnpm` 有版本号

---

## 六、装好之后你就能做什么

完成这份清单后，你的电脑就具备了下面这些能力：

- 跑前端 `Next.js`
- 跑后端 `FastAPI`
- 用 Docker 起数据库和缓存
- 用 `uv` 管 Python 环境
- 用 `pnpm` 管前端依赖
- 为后面的 OCR 能力做好准备

也就是说，这时已经可以正式开始《职点迷津》节点 1 的开发了。

---

## 七、如果你只想看最短版本

只做这 5 件事就够：

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
curl -LsSf https://astral.sh/uv/install.sh | sh
uv python install 3.11
corepack enable pnpm
corepack use pnpm@latest-10
```

然后手动安装并启动 Docker Desktop。

## 八、我建议的下一步

环境装完后，下一步就不是继续看文档了，而是直接做这 3 件事：

1. 初始化项目目录结构
2. 写 `docker-compose.yml`
3. 起 `PostgreSQL / Redis / MinIO`

这样我们就正式进入开发阶段了。
