fix: 修复AuthProvider使用useEffectEvent导致的hydration失败问题

根因：useEffectEvent在部分Next 16 + React 19组合下会导致客户端hydration失败，页面可见但按钮事件不绑定（按钮像"死的"）

修复方案：移除useEffectEvent，改用useEffect + async restore + cancel guard
- auth-provider.tsx: 移除useEffectEvent，恢复会话逻辑改为普通useEffect + async
- jobs/page.tsx: "去简历优化"改为先调后端创建会话再跳转
- match-report-panel.tsx: 按钮可用性门禁和提示文案完善

验证：cd apps/web && npm run lint/build通过

复测步骤：
1. 重启前端：cd apps/web && npm run dev（不要用旧进程热更新）
2. 浏览器强刷（Mac: Cmd+Shift+R）
3. 点击"新建目标岗位""创建 JD""更新匹配"应有即时UI变化
4. 报告成功后点"去简历优化"应跳转到带sessionId的优化页

风险：如仍"全部按钮无响应"，请提供浏览器Console首条红色报错
