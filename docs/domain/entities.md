# Domain Entities

## Resume

主简历是整个工作流的事实来源。

关键特征：

- 原始文件信息
- 解析状态
- Markdown 产物
- 结构化数据
- 版本信息

## Job Description

目标岗位不是纯文本输入框，而是后续所有定制与面试的上下文源。

关键特征：

- 原始 JD 文本
- 标题、公司、地点等基础字段
- 结构化要求与职责
- 版本信息

## Match Report

连接 resume 和 job 的桥梁层。

关键特征：

- fit band / score
- gap insights
- action pack
- tailoring plan
- interview blueprint

## Tailored Resume / Optimization Session

专属简历结果应被看作可继续使用的工作产物，而不是一次性文本建议。

关键特征：

- 来源 resume / job / report 版本
- 草稿 sections
- selected tasks
- optimized markdown
- 下载能力

## Mock Interview Session

模拟面试是带状态的持续训练流程。

关键特征：

- 来源岗位与简历上下文
- 当前 turn
- follow-up 决策
- review summary
- prep / retry / finish lifecycle

## Candidate Profile

候选人画像是横跨 resume、job、interview 的轻量长期记忆层。

关键特征：

- 偏好信息
- 目标方向
- 从工作流中沉淀的稳定事实
- 不与单次会话强绑定
